# 메시지 상태

모든 `Message` 는 다음 `status` 중 하나를 가집니다.

```
pending ──▶ streaming ──▶ done
              │     │
              │     └──▶ error
              └────────▶ aborted
```

| 상태        | 의미                                                        |
| ----------- | ----------------------------------------------------------- |
| `pending`   | 서버로 요청은 보냈으나 아직 첫 토큰이 오지 않음             |
| `streaming` | 최소 한 개 이상의 `StreamPart` 가 도착해 내용이 쌓이는 중   |
| `done`      | `finish` 이벤트로 정상 종료                                 |
| `error`     | 전송 계층이 에러를 던졌거나 `{ type: 'error' }` 파트를 받음 |
| `aborted`   | `stop()` 호출로 중단됨                                      |

## 컨트롤러 상태 (`ControllerStatus`)

`ChatController` 자체도 별도의 상태값을 가집니다.

- `idle` — 진행 중인 요청이 없음
- `submitting` — 요청 전송 직후 (첫 파트 대기)
- `streaming` — 스트리밍 중
- `error` — 직전 요청이 에러로 종료됨

`useChat` 의 `isStreaming` 은 `submitting | streaming` 일 때 `true` 입니다.

## Idle 타임아웃

컨트롤러는 **청크 간 최대 간격** 기반의 타임아웃을 제공합니다 (총 요청 시간이
아님). 긴 생성 작업 중에도, 일정 시간 새 청크가 오지 않으면 요청을
중단합니다.

```ts
useChat({
  transport,
  idleTimeoutMs: 30_000, // 30초 동안 청크가 없으면 abort
})
```

기본값은 60초입니다.
