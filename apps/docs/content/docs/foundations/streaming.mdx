# 스트리밍과 SSE

`@axe-ai-sdk/core` 는 Server-Sent Events 를 안전하게 파싱하는 `parseSSE` 함수를
제공합니다. `fetch` 응답 본문의 `ReadableStream<Uint8Array>` 를 넘기면
`{ event, data, id, retry }` 형태의 이벤트를 비동기 이터러블로 돌려줍니다.

```ts
import { parseSSE } from '@axe-ai-sdk/core'

for await (const event of parseSSE(response.body)) {
  console.log(event.data)
}
```

## 파서가 보장하는 것

- **CRLF / LF 동시 지원** — 스펙상 허용되는 두 개행 문자를 모두 처리합니다.
- **멀티라인 `data:`** — 여러 `data:` 라인이 이어지면 `\n` 으로 이어 붙입니다.
- **주석 (`:`) 무시** — keepalive 용 주석 라인은 건너뜁니다.
- **부분 청크 내성** — 임의의 바이트 경계에서 스트림이 잘려도 다음 청크에서
  이어서 파싱합니다.
- **UTF-8 안전성** — 멀티바이트 문자가 청크 경계에서 잘려도 글자가 깨지지
  않도록 `TextDecoder({ stream: true })` 를 사용합니다.

## StreamPart 로 변환

파싱된 SSE 이벤트를 그대로 UI에 넘기지 않고, 어댑터에서 `StreamPart` 유니언
타입으로 변환해 내보냅니다.

```ts
for await (const event of parseSSE(response.body)) {
  const parsed = JSON.parse(event.data)

  if (parsed.type === 'delta') {
    yield { type: 'text-delta', delta: parsed.content }
  } else if (parsed.type === 'tool_call') {
    yield { type: 'tool-call', toolCall: parsed.toolCall }
  } else if (parsed.type === 'done') {
    yield { type: 'finish', reason: 'stop' }
  }
}
```

컨트롤러는 이 파트들을 받아 `Message` 필드에 반영합니다. 자세한 내용은
[전송 계층](/docs/foundations/transports) 을 참고하세요.
