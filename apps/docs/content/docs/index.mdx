# axe-ai-sdk

React 애플리케이션에서 LLM 스트리밍 챗 UI를 만들기 위한 SDK 입니다. 전송
(transport) 계층이 프로토콜과 분리되어 있어, 어떤 백엔드를 쓰든 동일한 React
훅과 상태 기계를 사용할 수 있습니다.

<Callout type='info'>
  Vercel 의 `ai` SDK 에서 영감을 받았지만 **어댑터 중심** 설계입니다.
  백엔드(OpenAI, Anthropic, MCP, 커스텀 게이트웨이)는 `ChatTransport` 하나만
  구현하면 됩니다.
</Callout>

## SDK 가 해결하는 것

기존 챗 SDK 들은 보통 특정 백엔드(예: OpenAI 호환 API)나 특정 프레임워크와
강하게 결합되어 있습니다. 이 SDK 는 다음 문제를 해결합니다.

- **백엔드 종속 탈피** — `ChatTransport` 인터페이스 하나만 구현하면 OpenAI,
  Anthropic, MCP, 사내 게이트웨이 등 어떤 백엔드도 꽂을 수 있습니다.
- **스트림 안정성** — SSE 파서가 CRLF/LF, 멀티라인 `data:`, 코멘트, 임의
  바이트 경계에서 잘린 청크, UTF-8 스트림을 모두 안전하게 처리합니다.
- **요청 격리** — 요청별 `AbortController` 로 동시에 여러 메시지가 발사되어도
  스트림이 뒤섞이지 않습니다.
- **에이전트 / 툴 / RAG 일급 지원** — `StreamPart` 유니언이 `thinking-step`,
  `tool-call`, `tool-result`, `citation`, 임의 `metadata` 를 직접 전달합니다.

## 핵심 개념

| 개념             | 설명                                                                                 |
| ---------------- | ------------------------------------------------------------------------------------ |
| `ChatTransport`  | 요청을 받아 `StreamPart` 비동기 이터러블을 반환하는 어댑터                           |
| `ChatController` | 메시지 상태 기계. 전송 / 중단 / 재전송 / 타임아웃 관리                               |
| `StreamPart`     | 전송 계층이 내보내는 이벤트 단위 (텍스트 델타, 툴 호출 등)                           |
| `Message`        | 역할, 상태(`pending`·`streaming`·`done`·`error`·`aborted`), 메타데이터를 가진 메시지 |
| `useChat`        | React 훅. 컨트롤러를 래핑해 입력·메시지·상태를 제공                                  |

## 패키지 구성

- [`@axe-ai-sdk/core`](/docs/core/overview) — 전송 계층 독립 스트리밍, SSE 파서,
  `ChatController`, 에러, 타입. 런타임 의존성 없음.
- [`@axe-ai-sdk/react`](/docs/react/use-chat) — `useChat` 훅, 낙관적 UI,
  localStorage 영속화. React 19 피어 의존성.

#### 서버를 SSE 포맷으로 맞추기

`DefaultChatTransport` 가 이해하는 유일한 와이어 포맷은 다음 두 규약뿐입니다.

1. **`event:` 이름 = `StreamPart.type`** (`text-delta`, `thinking-step`,
   `tool-call`, `tool-result`, `citation`, `message-start`, `metadata`,
   `error`, `finish`)
2. **`data:` 는 JSON 객체** — 해당 `StreamPart` 의 나머지 필드

```
event: text-delta
data: {"delta":"안녕"}

event: thinking-step
data: {"step":{"agent":"planner","status":"running","thought":"의도 파싱 중"}}

event: citation
data: {"citation":{"id":"doc-1","title":"Docs","url":"https://..."}}

event: finish
data: {"reason":"stop"}
```

이미 OpenAI / Anthropic / 레거시 게이트웨이를 쓰고 있어서 포맷을 못 바꾼다면
BFF (Next.js route handler, Cloudflare Worker 등) 에서 한 번만 번역하거나,
`ChatTransport.send` 를 직접 구현하세요. 자세한 내용은
[DefaultChatTransport 페이지](/docs/core/default-transport) 참고.

#### `DefaultChatTransport` 구성

먼저 전송 계층을 만듭니다. 이 단계에서 **인증 헤더**, **커스텀 POST 바디**,
**디버그 훅**을 한번에 정의합니다.

```ts filename="lib/chat-transport.ts"
import { DefaultChatTransport, bearerFromCookie } from '@axe-ai-sdk/react'

// 여러 요청에서 재사용되도록 모듈 스코프에 한 번만 만듭니다.
export const transport = new DefaultChatTransport({
  api: '/api/chat',

  // 1) 인증: 쿠키에서 JWT 를 읽어 매 요청 Authorization 헤더로 주입.
  //    bearerFromCookie 는 함수를 반환하므로 토큰 갱신이 즉시 반영됩니다.
  //    다른 헤더와 섞고 싶다면 함수로 감싸세요.
  headers: () => ({
    ...bearerFromCookie('auth_token')(),
    'X-Tenant': getTenantId(),
  }),

  // 2) POST 바디: 기본값은 { messages, conversationId } 입니다.
  //    sessionId, 모델명, 시스템 프롬프트 등 서버가 기대하는 필드를 여기서 추가합니다.
  //    request.metadata 는 useChat.submit({ metadata }) 로 넘긴 값입니다.
  prepareBody: (request, state) => ({
    messages: request.messages,
    conversationId: state.conversationId,
    sessionId: getSessionId(),
    model: 'claude-opus-4-6',
    ...request.metadata, // per-request override
  }),
})

// 3) (선택) SSE 디버그 리스너 — 개발 중 네트워크 탭을 열지 않고도 청크를 볼 수 있습니다.
if (process.env.NODE_ENV !== 'production') {
  transport.onSSE((e) => console.debug('[sse]', e))
}
```

#### `useChat` 로 UI 연결

```tsx filename="components/chat.tsx"
'use client'
import {
  useChat,
  Markdown,
  type Message,
  type ThinkingStep,
  type ToolCall,
  type Citation,
} from '@axe-ai-sdk/react'
import '@axe-ai-sdk/react/styles.css'
import { transport } from '@/lib/chat-transport'

export function Chat() {
  const {
    // 상태
    messages,
    status, // 'idle' | 'submitting' | 'streaming' | 'error'
    isStreaming, // status === 'submitting' | 'streaming'
    error,

    // 입력 바인딩
    input,
    handleInputChange,
    handleSubmit,

    // 명령
    stop, // 현재 스트림 중단 → 메시지 상태 'aborted'
    reload, // 마지막 assistant 메시지 재생성 (regenerate)
    clear, // 메시지 + 영속화 초기화 (conversationId 유지)
    resetChat, // clear + conversationId + 전송 상태까지 리셋
  } = useChat({
    transport,

    // 로드 시점에 보여줄 웰컴 메시지
    initialMessages: [
      {
        id: 'sys-1',
        role: 'system',
        content: '당신은 axe-ai-sdk 문서 도우미입니다.',
        status: 'done',
        createdAt: Date.now(),
        updatedAt: Date.now(),
      },
    ],

    // localStorage 영속화. sanitize 는 기본 true 로 중간 스트리밍 상태를 저장하지 않습니다.
    persistence: { key: 'chat-demo', sanitize: true },
    conversationIdStorageKey: 'chat-demo:cid',

    // 스트림 파트가 idleTimeoutMs 동안 없으면 자동 abort 합니다. (0 = 비활성)
    idleTimeoutMs: 30_000,

    // 완료/에러 시 1 회 호출. 분석 이벤트를 여기서 보냅니다.
    onFinish: (m) => analytics.track('chat_message_done', { id: m.id }),
    onError: (err) => console.error('[chat]', err),
  })

  return (
    <div className='chat'>
      {messages
        .filter((m) => m.role !== 'system')
        .map((m) => (
          <MessageBubble key={m.id} message={m} />
        ))}

      {error && (
        <div role='alert' className='chat-error'>
          에러: {error.message}
          <button type='button' onClick={() => reload()}>
            다시 시도
          </button>
        </div>
      )}

      <form
        onSubmit={(e) =>
          handleSubmit(e, {
            // per-request 메타데이터 — prepareBody 의 request.metadata 로 전달됩니다.
            metadata: { temperature: 0.2 },
            // 유저 메시지에 함께 저장되는 메타데이터 (영속화 포함).
            userMetadata: { clientTs: Date.now() },
          })
        }
      >
        <textarea
          value={input}
          onChange={handleInputChange}
          placeholder='무엇이든 물어보세요'
          disabled={status === 'error'}
        />
        <div className='chat-actions'>
          {isStreaming ? (
            <button type='button' onClick={stop}>
              중지
            </button>
          ) : (
            <button type='submit' disabled={!input.trim()}>
              전송
            </button>
          )}
          <button type='button' onClick={() => reload()} disabled={isStreaming}>
            재생성
          </button>
          <button type='button' onClick={resetChat}>
            대화 초기화
          </button>
        </div>
      </form>
    </div>
  )
}

function MessageBubble({ message }: { message: Message }) {
  return (
    <article data-role={message.role} data-status={message.status}>
      <header>
        <strong>{message.role}</strong>
        {message.status === 'streaming' && <span>· 응답 중…</span>}
        {message.status === 'aborted' && <span>· 중단됨</span>}
        {message.status === 'error' && <span>· 오류: {message.error}</span>}
      </header>

      {/* 에이전트 사고 과정 — StreamPart: thinking-step */}
      {message.thinkingSteps?.map((s, i) => (
        <ThinkingRow key={s.id ?? i} step={s} />
      ))}

      {/* 툴 호출 — StreamPart: tool-call / tool-result */}
      {message.toolCalls?.map((t) => (
        <ToolRow key={t.id} call={t} />
      ))}

      {/* 본문 — 코드 블록·표·링크가 포함된 마크다운 안전 렌더링 */}
      <Markdown>{message.content}</Markdown>

      {/* RAG 인용 — StreamPart: citation */}
      {message.citations && message.citations.length > 0 && (
        <footer className='citations'>
          <ol>
            {message.citations.map((c) => (
              <CitationItem key={c.id} citation={c} />
            ))}
          </ol>
        </footer>
      )}
    </article>
  )
}

function ThinkingRow({ step }: { step: ThinkingStep }) {
  return (
    <div className='thinking'>
      <em>{step.agent}</em>
      {step.status === 'running' ? '…' : '✓'}
      {step.thought && <span>{step.thought}</span>}
    </div>
  )
}

function ToolRow({ call }: { call: ToolCall }) {
  return (
    <details data-status={call.status}>
      <summary>
        🔧 {call.name} ({call.status})
      </summary>
      <pre>{JSON.stringify(call.args, null, 2)}</pre>
      {call.result !== undefined && (
        <pre>{JSON.stringify(call.result, null, 2)}</pre>
      )}
      {call.error && <div role='alert'>{call.error}</div>}
    </details>
  )
}

function CitationItem({ citation }: { citation: Citation }) {
  return (
    <li>
      {citation.url ? (
        <a href={citation.url} target='_blank' rel='noreferrer'>
          {citation.title ?? citation.url}
        </a>
      ) : (
        (citation.title ?? citation.id)
      )}
      {citation.snippet && <blockquote>{citation.snippet}</blockquote>}
    </li>
  )
}
```

#### 서버 엔드포인트 준비

이 SDK 는 **클라이언트 측**만 다룹니다. 서버는 위 canonical 포맷으로 SSE
스트림을 내려주기만 하면 됩니다. Next.js App Router 예시는
[예제 디렉터리](https://github.com/kimcy/axe-ai-sdk/tree/main/examples/gateway)
를 참고하세요. 비표준 서버를 직접 어댑팅하는 예시는 같은 디렉터리의
[`gateway-transport.ts`](https://github.com/kimcy/axe-ai-sdk/blob/main/examples/gateway/src/gateway-transport.ts)
에 있습니다.
